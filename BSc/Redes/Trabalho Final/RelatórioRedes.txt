O servidor ira ser responsavel por gerir os dados enquanto o cliente podera consultar, editar ou ate mesmo inserir novos dados. Como exemplo, consideramos um sistema de lista telefonica, onde temos "clientes" e "numeros de telefone" associados. Para o cliente poder aceder aos dados do servidor, foi implementado um sistema de autenticacao, cujo e necessaria a introducao de um username e de uma password. Usamos a biblioteca do python pickle para guardar os dados em disco (ficheiro).Sistema Base a implementarNesta fase do trabalho foram implementadas varias funcionalidades para o sistema tais como a possibilidade de o cliente consultar, remover ou inserir novos dados.Para o cliente obter os numeros de um contacto associado a lista telefonica basta digitar “getphone NomeDaPessoa”. Para isso e implementada uma funcao getNumber (sock, nome) no servidor que ira receber o nome que o utilizador quer pesquisar na lista telefonica e ira mandar para o cliente todos os numeros associados aquele nome. No caso de o nome nao existir na lista telefonica, o cliente recebe uma mensagem “Error finding name”. Em vez de consultar atraves do nome, o cliente pode possuir um numero e desejar pesquisar os nomes que aquele mesmo numero esta associado. Para isso acontecer e implementada a funcao reverse (sock, numero) que ira pesquisar no dicionario (agenda telefonica), todos os nomes a qual o numero introduzido e associado. Para isso acontecer o cliente tera que introduzir na consola “getphone NumeroDaPessoa”.O sistema tambem possui a funcionalidade de introduzir novos contactos na lista telefonica. A funcao setNumber (sock, nome, numero) e implementada no servidor e ira verificar se o nome que o cliente introduziu ja esta presente na agenda. Caso este ja esteja e acrescentado o numero de telefone ao seu contacto. Caso nao esteja, e acrescentado uma Key no dicionario (Nome do contacto) com o numero associado a esse nome (Value). Se o numero ja estiver na agenda, e mandando um erro ao cliente “Erro setNumber” que ira decifrar a mensagem mandada pelo servidor e se for “Erro setNumber” e feito o print na consola de “Error setting number or name”. Para utilizador esta funcao e necessaria escrever na consola “getphone -set nome numero”.Por fim, atraves da funcao removeNumber (sock, nome, numero) o cliente pode remover clientes da lista telefonica atraves do comando “getphone -del nome numero” ou “getphone -del nome”. Ao utilizar o primeiro comando, e apagado da agenda telefonica o numero especifico aquele nome. Caso so exista aquele numero e apagado tambem o nome da pessoa da agenda. Caso o numero nao esteja relacionado com aquele nome e transmitido ao cliente a mensagem “ERRO removeNumber”. No caso o utilizador executar o comando “getphone -del nome” e apagado o nome do contacto da agenda telefonica juntamente com todos os numeros associados ao mesmo. Autenticacao 

O sistema de autenticacao e apenas para um utilizador usando como username : Admin e password : Admin, sendo assim o unico que tem poderes para modificar ou consultar a lista de contactos.
